# Day 8 — Работа с токенами и контекстным окном

Задание AI Advent Challenge #8, день 8. Ветка `task8` (поверх `task7` «Контекст»).

## Задание

Добавить в код агента подсчёт токенов:
- для текущего запроса;
- для всей истории диалога;
- для ответа модели.

Сравнить короткий / длинный / превышающий лимит диалоги и показать, **как растут
токены и стоимость** по ходу беседы и **что ломается при переполнении** контекстного окна.

## Что сделано

### 1. Подсчёт токенов — факт + оценка

Два источника, и это сознательно:

| Что считаем | Источник | Где в коде |
|---|---|---|
| Вся история (вход) | факт `prompt_tokens` из API | `UsageDto` → `TokenUsage` |
| Ответ модели | факт `completion_tokens` | то же |
| Текущий запрос (до отправки) | локальная оценка | `TokenEstimator` |

- **Факт `usage` больше не выбрасывается.** Раньше `AiAgent` отдавал только текст ответа;
  теперь `AgentAnswer.usage` пробрасывается до UI и сохраняется по сообщению в Room.
- **Локальная оценка до запроса** (`TokenEstimator`) — кириллице-aware эвристика
  (кириллица ~2 симв./токен, латиница ~4). Нужна, чтобы показывать заполнение окна и ловить
  переполнение **заранее** (API отдаёт цифры только постфактум). «Оценка vs факт» — учебная соль.
- Под каждым ответом агента — чип `вход N · ответ N · всего N ток.` (+ `кэш N ток.` при попадании
  в кэш промпта).

### 2. Контекстное окно: лимит, стоимость, переполнение

- У каждой модели (`AgentLlmModel`) появились **лимит окна** (`maxContextTokens`) и **тариф**
  (`ModelPricing`: вход / кэш-вход / выход за 1M токенов; бесплатные OpenRouter-модели → $0).
- **Демо-лимит окна** и **авто-обрезка истории** (`AgentConfig.demoContextLimitTokens`,
  `autoTrimHistory`) — настраиваются в чате (⚙ → «Контекст и токены»). Маленький лимит позволяет
  упереться в окно за пару сообщений и снять это на видео.
- При переполнении:
  - **авто-обрезка выключена** → детерминированная ошибка `AppError.ContextOverflow` (красный
    баннер в ленте, ответа нет) — *что ломается*;
  - **включена** → **sliding window**: самые старые сообщения отбрасываются (system prompt и
    последний вопрос сохраняются), агент отвечает, но «забывает» ранний контекст — *как чинится
    ценой памяти*.

### 3. Визуализация

- **Индикатор окна** над композером: `Контекст: used / limit ток.`, цвет синий → жёлтый (≥80%) →
  красный (переполнение), плюс счётчик `в окне N из M сообщений`.
- **Граница окна в ленте** (при авто-обрезке): сообщения, «выехавшие» из окна, приглушены с
  пометкой «вне контекста», на границе — разделитель «↑ модель это уже не видит». Граница
  считается тем же алгоритмом, что и реальная обрезка (`AiAgent.windowStartIndex`), и обновляется
  вживую с учётом черновика.
- **Вкладка «Статистика»** (заменила учебные «Документы»/«Готовность»): суммарные токены,
  вход/ответ/из кэша, оценка стоимости, график накопительного роста токенов по ходам и разбивка по
  диалогам (сравнение короткого и длинного).

## Поток работы агента

```
ChatScreen → ChatViewModel.sendMessage()
          → AiAgent.ask(config, conversation)
              // оценка токенов → проверка окна → (sliding window | ContextOverflow)
          → AiChatRepository → DeepSeekApi / OpenRouterApi (chat/completions)
          → AgentAnswer(content, usage) → история и токены в Room → UI
```

## Ключевые файлы

- `core/domain/.../agent/TokenEstimator.kt` — локальная оценка токенов.
- `core/domain/.../agent/AiAgent.kt` — подсчёт, проверка окна, sliding window, проброс `usage`.
- `core/model/.../ai/AgentModels.kt` — `maxContextTokens`, `ModelPricing`, `AgentConfig`.
- `core/data/.../ai/local/*` — токены по сообщению в Room (версия БД 3) + агрегат для статистики.
- `feature/chat/.../screen/ChatScreen.kt` — чип токенов, индикатор окна, граница окна, настройки.
- `app/.../ui/statistics/StatisticsScreen.kt` — дашборд токенов и стоимости.

## Как проверить / снять видео

Три отдельных диалога (кнопка диалогов → «Новый диалог»), сравнение на вкладке «Статистика».

1. **Короткий** — 2–3 вопроса; чип `вход` почти не растёт, стоимость копеечная.
2. **Длинный** — 8–10 ходов; `вход` растёт каждый ход (история переотправляется), график показывает
   сверхлинейный рост, стоимость заметно выше.
3. **Переполнение** — ⚙ лимит `1000`, авто-обрезка **вкл**: посади факт в начале, растяни диалог →
   старые пузыри гаснут («вне контекста»), появляется разделитель, счётчик `в окне N из M`; спроси
   про ранний факт — агент не вспомнит. Выключи авто-обрезку → красный индикатор и ошибка
   `ContextOverflow`.

## Заметки

- Лимиты окон и цены моделей в `AgentModels.kt` — **ориентир для демонстрации**; перед продом
  сверять с официальным прайсингом провайдера.
- Оценщик токенов — **эвристика**, а не BPE-токенайзер модели (это намеренно: показываем
  «оценка ≈ X / факт = Y»).
- Room пересоздаётся через `fallbackToDestructiveMigration` (dev), версия схемы поднята до 3 —
  миграция не пишется.

# AdventAI Project Context

Ты Senior Android Developer и Product Designer уровня 2026 года. Работаешь над Android-приложением AdventAI.

## Цель проекта

AdventAI - Android-приложение на Jetpack Compose для работы с AI-агентами.

Текущий главный сценарий:
- главный экран со списком агентов;
- выбор агента;
- чат с агентом;
- настройки агента через bottom sheet.

Агент должен быть отдельной сущностью, а не просто вызовом API из UI. Логика запроса к LLM инкапсулируется в доменном слое.

## Технологии

- Kotlin
- Jetpack Compose
- Material 3
- Hilt
- Retrofit / OkHttp
- Gradle Kotlin DSL
- DeepSeek / OpenRouter API

## Архитектура

Используй существующие модули:

- `app` - MainActivity и навигация.
- `feature:home` - главный экран и список агентов.
- `feature:chat` - экран диалога и настройки агента.
- `core:model` - модели данных.
- `core:domain` - use case и логика агента.
- `core:data` - repository implementation.
- `core:network` - API clients.
- `core:designsystem` - тема, цвета, общие UI-компоненты.

Не ломай модульные границы. UI не должен напрямую знать детали HTTP-запросов.

## Текущий UI

Главный экран:
- горизонтальный carousel агентов на `LazyRow`;
- карточка в центре увеличивается при скролле;
- используется `rememberSnapFlingBehavior`;
- стиль сине-белый;
- есть карточка `Визовый специалист`;
- есть карточка `Добавить агента`;
- старые дни заданий не отображаются.

Экран чата:
- кастомная верхняя панель внутри `ChatScreen`;
- панель учитывает `statusBarsPadding()`;
- кнопка назад слева;
- название агента в центре/слева;
- иконка настроек справа сверху;
- поле ввода закреплено снизу;
- `imePadding()` применяется к нижнему composer, а не ко всему экрану;
- настройки открываются bottom sheet.

## Дизайн-система

Общий стиль: современный, чистый, продуктовый, сине-белый.

Правила:
- основные цвета брать из `MaterialTheme.colorScheme`;
- избегать hardcoded цветов;
- не использовать разноцветную палитру без причины;
- все карточки и цветные блоки должны иметь скругления;
- цветные зоны внутри карточек должны быть clipped через `RoundedCornerShape`;
- тексты не должны вылезать из контейнеров;
- использовать `maxLines` и `TextOverflow.Ellipsis`, где текст может быть длинным;
- стандартные иконки делать через vector drawable / Material-style, а не самодельный Canvas.

## Правила работы

Перед изменениями:
- прочитать существующие файлы;
- понять текущую архитектуру;
- не переписывать логику, если задача только про дизайн;
- не возвращать старые дни заданий в навигацию;
- не трогать API-логику без явного запроса.

После изменений запускать:

```powershell
.\gradlew.bat :app:compileDebugKotlin --console=plain
```

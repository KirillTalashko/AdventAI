### Продукт
AdventAI — это **продуктовое** приложение с AI-агентом (не учебный пример). Агент-«Визовый специалист»:
- **ищет информацию** по визам, странам, правилам и требованиям;
- **работает с документами** по визам (состав пакета, анкеты, проверка, чек-листы);
- **готовит клиента к подаче**: сроки, риски отказа, финансы, следующие шаги.
UX, дизайн и надёжность важны — это продукт, которым пользуется реальный человек, готовящийся к подаче.

### Стек и архитектура (не ломать)
Kotlin · Jetpack Compose · Material 3 · Hilt · Retrofit/OkHttp · Room · Gradle Kotlin DSL ·
DeepSeek / OpenRouter API. Модульная clean architecture: `app` / `feature:*` / `core:*`.
Агент — отдельная доменная сущность `AiAgent`; UI не знает про HTTP; сетевые детали в `core:network`;
история диалога — в Room (`ChatHistoryRepository`). Подробности и карта модулей — в `CLAUDE.md` и
`.claude/PROJECT_CONTEXT.md`, `.claude/SCREEN_REQUIREMENTS.md`, `.claude/AI_AGENT_TOOLS_AND_LLM_NOTES.md`.

### Дизайн-система (`.claude/DESIGN_SYSTEM.md`)
Сине-белый стиль, скруглённая геометрия, цвета только из `MaterialTheme.colorScheme` (без hardcode),
адаптивность, `maxLines`/`Ellipsis`, безопасные зоны. Без новых случайных цветов и стандартных
прямоугольных Material-компонентов поверх продуктовых скруглённых поверхностей.

### Формат челленджа
AI Advent Challenge #8. Каждый будний день — задание «День N». Сдача = **видео + код**. Реализуем
**поэтапно** (общая память/контекст агента доводим к дню 10). Описание результата в задании —
примерное; важен работающий результат, подтверждённый демонстрацией.

### Как работай над заданием (каждый день)
1. **Сначала разберись:** прочитай `CLAUDE.md` + нужные `.claude/*.md` и релевантный код. Не выдумывай —
   проверяй по коду и фактам.
2. **Свяжи с продуктом:** объясни, что это задание даёт клиенту визового агента.
3. **Покажи краткий план** до кода: что меняем, какие файлы, какие решения/развилки. Если есть выбор
   (хранилище, библиотека, объём фичи, UX) — задай мне 2–4 варианта с рекомендацией, не угадывай молча.
4. **Переиспользуй существующее** (`AiAgent`, репозитории, use case'ы, дизайн-систему). Не ломай
   модульные границы, не трогай API-логику без нужды, не возвращай старые «дни» в видимую навигацию.
5. **Реализуй** по дизайн-системе и архитектуре.
6. **Проверь:** `.\gradlew.bat :app:compileDebugKotlin --console=plain`. По возможности запусти на
   эмуляторе и проверь реальное поведение (не только компиляцию). Если что-то не работает — диагностируй
   причину (код vs окружение), не гадай; у эмулятора бывает залипает сеть → cold boot
   (`emulator -avd <name> -dns-server 8.8.8.8 -no-snapshot-load`).
7. **Отчитайся честно:** что сделано, что проверено, что осталось; приложи доказательство (лог/скриншот).
8. **Коммит / push / PR и запись демо — ТОЛЬКО по моей явной команде.** Я сначала проверяю сам.
   Workflow: ветка `taskN` → PR в `master` (как task5/task6), коммиты в стиле «Day N. Title.».

## Как сделать из этого slash-команду `/day` (необязательно)

Чтобы не вставлять блок руками каждый день, можно завести команду Claude Code:
создать файл `.claude/commands/day.md`, положить туда тот же текст, а вместо
`<< вставь сюда текст задания >>` поставить `$ARGUMENTS`. Тогда вызов:

```
/day День 8. <текст задания>
```

Скажи — и я создам эту команду.
